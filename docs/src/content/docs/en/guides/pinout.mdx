## Configuration Flags

The configuration is divided into two groups: the pins that control the **direction of the motor** and the pins that **enable the H-bridge** of each motor.  
Only **one option per group** should be active at a time.

### Motor direction pins

```cpp
// #define USE_PWM_MOTOR_PIN      // Use PWM pins for motor direction
// #define USE_DIGITAL_MOTOR_PIN  // Use digital pins for motor direction
```

### Enable pins

```cpp
// #define USE_PWM_ENABLE_PIN     // Use PWM pins for motor enable
// #define USE_DIGITAL_ENABLE_PIN // Use digital pins for motor enable
```

<Aside type="note">
Uncomment only the option you need for each group. For example, if you want PWM motor direction and digital enable, you would uncomment:

```cpp
#define USE_PWM_MOTOR_PIN
#define USE_DIGITAL_ENABLE_PIN
```
</Aside>

## Logic Table: Which Pins Activate per Configuration

| Configuration           | ESP32 Pins Used                   | Arduino Pins Used    | Description                                 |
|-------------------------|-----------------------------------|----------------------|---------------------------------------------|
| Motor direction PWM     | GPIO 27, 14, 17, 16               | 9, 6, 5, 3           | IN1 ... IN4 controlled via PWM              |
| Motor direction DIGITAL | GPIO 27, 14, 17, 16               | 9, 6, 5, 3           | IN1 ... IN4 controlled via digital ON/OFF   |
| Enable PWM              | GPIO 32, 23                       | 10, 11               | Dedicated PWM pins for speed/enable control |
| Enable DIGITAL          | GPIO 32, 23                       | 10, 11               | Dedicated digital ENABLE pins               |
